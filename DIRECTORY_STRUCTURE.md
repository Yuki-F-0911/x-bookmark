# x-bookmark-digest プロジェクト構造と機能解説

現在の当プロジェクトは、情報収集（Xブックマーク、RSS、Web検索）から、内容の分析・要約、最終的な記事（ニュースレターやnote記事）の生成までを自動化するためのデータパイプラインとして機能しています。

以下に現在のディレクトリ構造と、それぞれのファイル・フォルダの役割をまとめました。

## 📁 ディレクトリ構造概略

```
x-bookmark-digest/
├── src/                # ダイジェスト処理の中核ロジック
├── scripts/            # 自動化パイプラインや単発実行用スクリプト群
│   └── naruse/         # 「成瀬さんのnote題材」に特化した独自パイプライン群
├── scheduler/          # Windowsタスクスケジューラー連携用スクリプト
├── tests/              # 単体テストコード
├── slack_gateway/      # Slack統合用のCloudflare Workers等
├── services/           # 音声文字起こし等、その他の拡張サービスモジュール
├── products/           # 将来のパッケージ化・商材化（Micro-SaaS等）に向けたドキュメント
├── micro-saas/         # Micro-SaaS展開向けのリソース（AnalysisAudit等）
├── newsletters/        # 生成されたニュースレターや記事の出力先（自動生成）
├── data/               # 一時データなどの保存場所
└── docs/               # プロジェクト設計や仕様の手引などのドキュメント
```

## 🛠️ 主要なフォルダとスクリプトの役割

### 1. `scripts/`（自動化・情報収集スクリプト）

日常的な情報収集や、自動化パイプラインのエントリーポイントです。

- **`full_pipeline.py`**
  情報収集 → 要約 → ニュースレター生成の手順を1つのコマンドで通して実行できる統合スクリプト。
- **`watcher.py`**
  ローカルの`Downloads`フォルダを監視し、Chrome拡張機能から出力されたCSV形式のブックマークを発見次第、ローカルで処理・補完してGitリポジトリ（`bookmarks.json`）へコミット・Pushする常駐監視ツール。
- **`fetch_tweets.py`**
  XのSyndication/oEmbed APIを叩いて、ブックマークしたツイート本文（長文含む）を自動取得し補完するスクリプト。
- **`auto_fetch_business_articles.py`**
  Web検索や人気サイトなど複数のソースから、指定されたビジネスキーワードの関連記事を自動収集する。
- **`fetch_with_websearch.py`**
  Claude CodeのWebSearch機能を想定し、定期的にトレンドを検索して結果をJSON保存する仕組みのプロトタイプ。
- **`generate_newsletter.py`**
  収集・要約した内容（`digest_cache.json`）をもとに、AIを活用してNote/Substack投稿向けのリッチな記事Markdownを自動生成する。

### 2. `scripts/naruse/`（成瀬さん向けパーソナライズ）

通常のツール活用系ブックマーク収集とは別に、成瀬さんの思想やパーソナリティに特化して「社会変革・組織論・パラダイムシフト」などを重点的に深掘りするためのパイプライン。

- **`integrated_daily_pipeline.py` / `daily_business_themes_pipeline.py`**
  手動ブックマークとWebやRSSから自動取得した記事を合体させ、抽出・スコアリングパイプラインを実行してnote題材候補を出力する日次スクリプト。
- **`extract_business_themes.py`**
  記事やブックマークを成瀬さんの志向（Will Out, 世界を一つの家族に 等）に合わせてスコアリングし、親和性の高い順にMarkdownでリスト化する分類エンジン。
- **`fetch_business_topics.py` / `fetch_business_topics_v2.py`**
  RSSフィードとWeb検索を組み合わせ、指定したビジネスキーワードに特化した情報を効率よく取得するためのスクリプト。
- **`naruse_complement_search.py`**
  単なる「AIツール紹介」に偏らないよう、社会変革・キャリア観・組織変革などのテーマを重点検索し、成瀬さんが書くべき高質なテーマを補完するスクリプト。

### 3. `src/`（コアロジック）

プロジェクトのエンジン部分。`scripts/` 内の各処理から呼び出される本質的な処理群です。

- **`bookmark_loader.py`**: CSV/JSON形式のブックマークの読み込み、解析、テキスト補完。
- **`summarizer.py`**: Claude/Gemini API等を利用し、抽出した記事内容から数行の要旨をまとめ上げるロジック。
- **`slack_notifier.py` / `slack_bot.py`**: まとまった要約データを、Slackチャンネルに対してきれいに整形して通知・送信するためのモジュール。
- **`search_enricher.py`**: 情報が足りない記事への追加のメタデータ取得。
- **`main.py`**: コアロジックを直接呼び出す際のエンドポイント。

### 4. `scheduler/`（自動実行の設定）

Windows環境のバックグラウンドで動作させるための起動用ファイルです。

- **`start_watcher.bat`**: `watcher.py` を呼び出すバッチスクリプト。
- **`setup_task_scheduler.ps1`**: バッチスクリプトをスタートアップ時に見えない状態（`run_hidden.vbs`利用）で常駐実行するよう、Windowsタスクスケジューラーに自動登録するインストール用スクリプト。

### 5. `slack_gateway/` & `services/`

- **`slack_gateway/`**
  Slack Event等をトリガーにして外部から処理を受け付けるためのCloudflare Workers用環境。
- **`services/`**
  将来的に音声文字おこし (`voice_to_article`) や 社内RAG (`corporate_rag`) などの独立した機能サービスを提供するための拡張モジュール用フォルダ。

### 6. `products/`

現在のツール群を将来的にパッケージ化し、販売・展開できるように整理したドキュメント類（「自動キュレーションパッケージ」「ミーティングから記事生成パッケージ」などのコンセプト案）が格納されています。

## 🎯 このプロジェクトができることの全体像

1. **完全非同期＆全自動の情報整理**
   ユーザーはXで気になったものをブックマーク保存用の拡張機能などでCSVに落とすだけ。あとは `watcher.py` が検知・同期し、本文の文字制限も回避して要約を作ります。
2. **パーソナライズされた記事の提案**
   `naruse/` ディレクトリにあるように、単なるサマリーツールではなく「特定の思想（ペルソナ）に合致した情報だけを高く評価し、noteの題材として提案する」というコンテキスチュアルなレコメンド機能を持っています。
3. **継続的なコンテンツ出力**
   `full_pipeline.py` を動かすことで、毎日の情報収集を「Slackへの共有」だけでなく「そのまま配信に使えるニュースレターやnote記事の下書き」に仕上げるコンテンツ自動生成工場として機能します。
