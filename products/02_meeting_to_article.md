# 🎙️ 会議音声→プロ記事変換プロンプト集

> Zoom・Teams・Google Meetの録音から、プロライター品質の記事を自動生成。
> 「話しただけ」で高品質コンテンツが完成します。

## この商品で得られるもの

### 1. VTT→構造化テキスト変換プロンプト

会議録音のVTTファイル（字幕形式）を読み込み、以下を自動処理：
- 相槌・フィラー（「えーと」「あのー」）の除去
- 話者ごとの発言分離
- 核心的主張の抽出と再構成

### 2. パーソナリティベース記事生成プロンプト

話者の特徴（語り口、専門分野、価値観）を事前定義し、
その人「らしい」文体で記事を自動生成します。

```
## パーソナリティ定義テンプレート

話者名: {名前}
専門分野: {分野}
語り口の特徴: {例：「〜だと思うんですよね」という語尾が多い}
大切にしている価値観: {例：人と人のつながり、テクノロジーの民主化}
よく使う比喩・表現: {例：「橋を架ける」「種をまく」}
```

### 3. 記事構成テンプレート（5種類）

| テンプレート | 用途 | 文字数目安 |
|-------------|------|-----------|
| インタビュー形式 | 対談・インタビュー記事 | 3000-5000字 |
| コラム形式 | 1人の主張をまとめる | 2000-3000字 |
| レポート形式 | 会議・勉強会のまとめ | 1500-2500字 |
| 対話形式 | 複数人の議論を再構成 | 2500-4000字 |
| SNS投稿用 | Twitter/Note用の短文 | 300-500字 |

### 4. 品質チェックプロンプト

生成された記事を以下の観点で自動レビュー：
- 事実の歪曲がないか
- 話者の意図と記事の論旨が一致しているか
- 読みやすさ（PREP法準拠）
- SEO最適化（タイトル・見出し・メタディスクリプション）

## プロンプト詳細

### VTT解析プロンプト

```
以下はZoom会議の文字起こし（VTT形式）です。
この録音内容を分析し、以下の形式で構造化してください。

## タスク
1. 各話者の発言を特定し、名前をラベリング
2. 相槌・フィラー・無関係な雑談を除去
3. 各話者の核心的な主張を抽出
4. 議論の流れを時系列で整理

## 出力形式
### 参加者
- {話者A}: {役割・立場}
- {話者B}: {役割・立場}

### 核心的主張
#### {話者A}
1. {主張1}（根拠: {発言の要約}）
2. {主張2}

### 議論の流れ
1. {テーマ1}: {概要}
2. {テーマ2}: {概要}
```

### パーソナリティベース記事生成プロンプト

```
以下の構造化された議論データと話者のパーソナリティ情報をもとに、
{話者名}の視点を中心としたnote記事を生成してください。

## パーソナリティ情報
{パーソナリティ定義}

## 記事要件
- 読者: {ターゲット読者}
- テーマ: {記事テーマ}
- 文字数: {目安}
- トーン: {話者}らしい語り口を再現しつつ、読みやすく整理
- 構成: リード → 本論（3セクション）→ まとめ

## 重要ルール
- 話者が実際に言っていないことを創作しない
- 核心的主張を軸に、補足説明を自然に織り込む
- 読者に「行動」を促す結びにする
```
